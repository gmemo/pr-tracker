## üèóÔ∏è BUILD & DEPLOYMENT WORKFLOW

### **Building the PWA** (IMPORTANT!)

Always use this command to build for production:
```bash
npm run build:web
```

**What this does:**
1. Runs `npx expo export --platform web` to build the app
2. Copies PWA assets from `public/` to `dist/`:
   - `manifest.json` - PWA configuration
   - `icon-192.png` and `icon-512.png` - App icons
   - `apple-touch-icon.png` - iOS home screen icon
3. Updates `dist/index.html` with proper manifest and icon links

**DO NOT use** `npx expo export --platform web` directly - it won't include PWA assets!

### **PWA Assets Location**

All PWA assets are in the `public/` folder:
- `public/manifest.json` - PWA config (name, colors, display mode)
- `public/icon-192.png` - 192√ó192 icon for Android
- `public/icon-512.png` - 512√ó512 icon for high-res displays
- `public/apple-touch-icon.png` - 180√ó180 icon for iOS
- `public/favicon.png` - Browser tab icon

These are automatically copied to `dist/` during build.

### **Deployment to Vercel**

**Automatic deployment:**
```bash
# Make changes to your code
git add .
git commit -m "your message"
git push

# Vercel automatically detects push and deploys!
# Wait 2-3 minutes, then check your Vercel URL
```

**Manual deployment via CLI:**
```bash
# Install Vercel CLI (first time only)
npm install -g vercel

# Deploy to production
vercel --prod
```

**Build settings (already configured in vercel.json):**
- Build Command: `npm run build:web`
- Output Directory: `dist`
- Framework: None (custom)

### **Docker Deployment (Optional)**

Docker container has been **removed to save resources**, but config is preserved in repo.

To run Docker locally again:
```bash
# Build and start
npm run build:web
docker build -t pr-tracker .
docker compose up -d

# Access at http://localhost:3000

# Stop and clean up when done
docker compose down
docker rmi pr-tracker
```

**When to use Docker:**
- Testing local deployment
- Self-hosting on your own server
- Sharing with users on local network
- Development environments

**Why we removed it:**
- Vercel handles public deployment (better!)
- Saves disk space (~5-10 MB)
- No need to rebuild container for every change
- Can easily rebuild when needed

## üÜò TROUBLESHOOTING GUIDE

### Common Issues

**Issue**: PWA not updating after changes
**Fix**: Hard refresh (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows)

**Issue**: Data not persisting
**Fix**: Check browser storage settings, ensure localStorage enabled

**Issue**: Import fails with error
**Fix**: Validate JSON structure, check for corruption

**Issue**: Docker container won't start
**Fix**: Check port 3000 not in use: `lsof -i :3000`

**Issue**: Vercel build fails
**Fix**: Check build logs, ensure `dist/` folder created

## ü§î DECISION LOG

Key architectural decisions made:
1. **No backend** - Privacy and offline-first approach
2. **React Native + Expo** - Cross-platform code reuse
3. **AsyncStorage** - Simple local storage, no DB overhead
4. **Context API** - Lightweight state management, avoid Redux complexity
5. **PWA first** - Fastest path to market, lowest barrier to entry
6. **Vercel hosting** - Free, reliable, great DX
7. **Donation model** - Keep free, optional support
